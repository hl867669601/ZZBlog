并发编程

发表于 2022-11-18 更新于 2022-11-20 分类于 并发编程

基本理论

并发三要素

可见性：一个线程对共享变量的修改，另外一个线程能够立刻看到。

原子性：一个操作或者多个操作 要么全部执行并且执行的过程不会被任何因素打断，要么就都不执行。

有序性：程序执行的顺序按照代码的先后顺序执行。

导致问题的原因：

  • CPU 增加了缓存，以均衡与内存的速度差异；// 导致 可见性问题

  • 操作系统增加了进程、线程，以分时复用 CPU，进而均衡 CPU 与 I/O 设备的速度差异；// 导致 原子性问题

  • 编译程序优化指令执行次序，使得缓存能够得到更加合理地利用。// 导致 有序性问题

Java内存模型

volatile可以保证可见性。

synchronized和lock可以用来保证有序性和原子性。

Happens-Before原则

  1. 单一线程原则：在一个线程内，前面的操作先行发生于后面的操作。
  2. 管程锁定原则：一个unlock操作先行发生于后面对同一个锁的lock操作
  3. volatile变量规则：对一个volatile变量的写操作先行发生于后面对这个变量的读操作。
  4. 线程启动规则：Thread对象的start()操作先行发生于此线程的每个操作。
  5. 线程加入规则：Thread对象的结束先行发生于join()方法返回。
  6. 线程中断规则：对线程interrupt()方法的调用，先行发生于被中断线程的代码检测到中断事件的发生。
  7. 对象终结规则：一个对象初始化完成（构造函数执行结束）先行发生于它的finalize()方法的发生
  8. 传递性：A先行发生于B，B先行发生于C，那么A先行发生于C

线程安全的方法

互斥同步

synchronized和ReentrantLock锁

非阻塞同步

无同步方案

栈封闭：多线程访问局部变量时，不会有线程安全问题，因为局部变量在虚拟机栈中，属于线程私有

ThreadLocal：每个线程都有一个专属于线程的存储空间。

锁优化

锁升级

现在JDK对Synchronized进行了优化，有一个锁升级的过程。

无锁->偏向锁->轻量级锁->重量级锁

自旋锁

自旋锁是一种乐观锁，认为一般不会产生锁竞争，它通过CAS调用加锁，如果失败了它不直接阻塞，而是停一小会再加，这样当同步资源锁定的时间很短时，性能会很好。当超过一定次数还是失败才会挂起。

锁消除

逃逸分析，如果明显不会逃逸出去被其它线程访问到，JVM会把他们当栈上数据对待，认为是线程独有的，这样就不会进行加锁。

锁粗化

如果JVM检测到对同一个对象连续加多次锁，那会对加锁的范围扩展到这一连续操作的外部，即只需加一次锁。

轻量级锁、偏向锁、重量级锁

Mark Word

HotSpot虚拟机的对象在内存中可以分为：对象头、实例对象和对齐填充。

而对象头又包含了MarkWord和所指向的类指针

MarkWord的后三位是与加锁相关的。

轻量级锁

加轻量级锁过程：

  1. JVM在当前线程栈帧中，创建一个锁记录（Lock Record）空间。
  2. 复制MarkWord到锁记录空间中。
  3. 用CAS操作将MarkWord修改为指向锁记录的指针。
  4. 如果更新成功就说明获取到了该对象的锁，然后跟新标志位为00。
  5. 如果更新失败，就会自旋，到一定次数后就要锁升级为重量级锁，这时标记为会变为10。

解锁：

  1. CAS操作替换Mark Word回到对象头中。
  2. 如果成功就解锁成功。
  3. 如果失败就说明存在竞争，锁又会膨胀

重量级锁

monitor对象：

每个Java对象都可以对应一个Monitor对象

加重量级锁过程：

  1. 一开始Monitor中的Owner为null
  2. 当加重量级锁的是时候，会将对象markword指向Monitor，然后Monitor的所有者Owner置为加锁的线程
  3. 当其它线程加锁时，会被放到EntryList进入阻塞状态
  4. WaitSet用来存放之前获得过锁，但不满足条件的线程（执行wait等方法）。

偏向锁

在大多实际环境下，锁不仅不存在多线程竞争，而且总是由同一个线程多次获取。为了减少不必要的CAS操作，会在对象头和栈帧的锁记录存储偏向的线程ID，加锁只需要测试对象的MarkWord是否指向当前线程，如果成功就表示拿到了锁。

如果不成功就会进行偏向锁的撤销。

CAS原理

CAS的全称为Compare-And-Swap，直译就是对比交换。是一条CPU的原子指令，其作用是让CPU先进行比较两个值是否相等，然后原子地更新某个位置的值。

简单解释：CAS操作需要输入两个数值，一个旧值(期望操作前的值)和一个新值，在操作期间先比较下在旧值有没有发生变化，如果没有发生变化，才交换成新值，发生了变化则不交换。

AQS

AQS是通过修改一个state状态来表示同步状态，然后用一个FIFO双向队列来进行线程排队。

AQS是个抽象类，要用AQS实现同步就需要继承或内置一个AQS

AQS定义了几个public方法，用于加共享锁和排他锁。里面调用了几个protected方法，子类就是通过重写这几个方法的逻辑来实现线程同步。
