Java.IO

File

File可以表示文件或目录信息，主要用的API: listFiles()、isDirectory()、exists()、mkdir()等。它们都不需要捕获异常。

InputStream

InputStream的常用API有：

  • read(): 返回输入流中下一个字节的数据。返回的值介于 0 到 255 之间。如果未读取任何字节，则代码返回 -1 ，表示文件结束。
  • read(byte[] b): 从输入流中读取一些字节存储到数组 b 中。如果数组 b 的长度为零，则不读取。如果没有可用字节读取，返回 -1。如果有可用字节读取，则最多读取的字节数最多等于 b.length ， 返回读取的字节数。这个方法等价于 read(b, 0, b.length)。
  • read(byte b[], int off, int len) ：在read(byte b[ ]) 方法的基础上增加了 off 参数（偏移量）和 len 参数（要读取的最大字节数）。
  • skip(long n) ：忽略输入流中的 n 个字节 ,返回实际忽略的字节数。
  • available() ：返回输入流中可以读取的字节数。
  • close() ：关闭输入流释放相关的系统资源。

Jdk9后又增加了:

  • readAllBytes() ：读取输入流中的所有字节，返回字节数组。
  • readNBytes(byte[] b, int off, int len) ：阻塞直到读取 len 个字节。
  • transferTo(OutputStream out) ： 将所有字节从一个输入流传递到一个输出流。

Java.NIO

基础概念：

  • I/O: 以流方式处理数据，一次处理一个字节数据

  • NIO: 以块方式处理数据，一次处理一个数据块，块被读入内存之后就是一个byte[]

  • Channel : Channel 是对原 I/O 包中的流的模拟，可以通过它读取和写入数据。

  • 缓冲区: 向通道发送或读取数据，都需要先经过缓冲区。

Buffer

buffer里有三个变量：

capacity：容量(不变)

limit: 可读写到的位置

position: 当前读写位置

写模式：

初始时position在0，limit在capacit位置

写多个字节后，position变成下一个要写的位置，limit不变。

读模式：

读之前要调用flip()

这时position会变到0位置，limit会变到原Postion位置，这样他们之间就是可以读的数据了。

调用clear方法

清空缓存区，postion就会回到0，limit回到最后。

Compact

compact 方法，是把未读完的部分向前压缩，然后切换至写模式

ByteBuffer

常用语句：

mark 和 reset

mark 是在读取时，做一个标记，即使 position 改变，只要调用 reset 就能回到 mark 的位置
